# Wedding Studio UX Skill

Use this project skill guide before making visual, interaction, or product-surface changes to Wedding Flow Studio.

## Product Standard

Wedding Flow Studio must feel like a premium visual wedding-day production studio. It should not feel like a generic dashboard, checklist, marketplace, spreadsheet, or text-heavy magazine layout.

## First Viewport Rule

Every primary route should open with:

- One clear focal surface.
- One obvious next action.
- At most three visible supporting panels.
- Minimal badges and metrics.
- Secondary reasoning hidden in drawers, details blocks, or deeper modules.

If the first viewport contains many cards, many equal-weight headings, multiple competing CTAs, or dense explanatory text, simplify before adding features.

## Visual Asset Rule

Use images only when they make the product easier to understand or emotionally stronger. Prefer one premium, calm, scene-setting asset over many decorative visuals.

Good assets:

- Venue atmosphere.
- Ceremony or reception mood.
- Texture that supports a planning canvas.
- Scene background that makes the wedding day feel previewable.

Avoid:

- Stock-looking couples.
- Cartoon wedding symbols.
- Hearts, rings, confetti, or cliché decoration.
- Busy images behind important text.
- Fake testimonials or unsupported claims.

## Screenshot Review

After frontend work:

1. Open the affected route in the browser.
2. Capture or inspect the first viewport.
3. Check for overlap, text density, weak hierarchy, too many cards, harsh color, and unclear next action.
4. If the screenshot still looks cluttered, remove elements before polishing details.

## Design Gate

Before final reporting, confirm:

- The first thing a user sees is the product experience, not explanation.
- The page has a calm visual rhythm.
- The primary action is obvious within three seconds.
- The color system feels premium and restrained.
- The user can understand what to do next without reading every block.
